#🏋️‍♀️ Interactive Fitness Tracking Dashboard --
A fully interactive Fitness Tracking Dashboard built using HTML, CSS, and JavaScript and Chart.js for Data Visualisation — designed to transform fitness data into meaningful, easy-to-understand insights.
This project focuses not only on visualizing data, but also on creating a clean user experience that helps users quickly understand trends, habits, and performance.

📊 Project Overview --
The dashboard combines multiple fitness metrics, interactive charts, and tracking components into a single responsive interface.
Instead of simply displaying numbers, the goal was to build a dashboard that highlights:
progress,
patterns,
correlations,
and actionable insights.
Every design decision — from color-coded indicators to chart layouts — was made with usability and clarity in mind.
✨ Features
📌 Live Summary Cards
Track important health metrics through dynamic cards:
🚶 Steps
🔥 Calories Burned
⏱️ Active Time
💧 Water Intake
😴 Sleep Hours

Each card includes:
- percentage change badges,
- trend indicators,and visual cues to quickly identify improvement or decline.
- 
📈 Interactive Charts (Chart.js) --
The dashboard includes multiple responsive data visualizations built using Chart.js:
- Weekly Step Count vs 10,000-Step Goal
- Daily Calorie Burn Trend
- Sleep & Recovery Analysis
- Workout Type Distribution (Donut Chart)
- Water Intake vs Active Minutes (Dual-Axis Chart)
These charts are designed to reveal patterns and relationships instead of just presenting raw numbers.

🔽 Dynamic Period Filter--
Users can switch between:
Today
Week
Month
The entire dashboard updates instantly, including:
metric cards,
charts,
and workout records.
This was implemented using pure JavaScript without any frameworks.

📋 Workout History Table--
A detailed workout tracking table is included with:
workout type,
heart rate,
calories burned,
distance covered,
and session details.
This acts as the supporting data layer behind the dashboard visualizations.

🧠 Key Design Insight
One of the most interesting realizations while building this project was:
* Sleep is a leading indicator, not a lagging one.
* Good sleep often predicts next-day performance, recovery, and activity levels.
That insight influenced the layout design and gave sleep tracking more visual importance within the dashboard.

*Another major focus was reducing cognitive load by using:
clean spacing,
accessible contrast,
clear visual hierarchy,
goal reference lines,
and color-based trend indicators.
